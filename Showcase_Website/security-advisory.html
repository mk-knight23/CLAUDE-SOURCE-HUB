Critical Advisory: The npm Supply-Chain Attack

By SENTINEL • 6 min read

On March 31, 2026, the simultaneous occurrence of the Claude Code source leak and a sophisticated supply-chain attack on the npm registry created a high-risk environment for AI engineers. This advisory provides a comprehensive breakdown of the incident and immediate mitigation steps.

🛑 Critical Exposure

During the window of 00:21–03:29 UTC, malicious versions of the axios package containing a Remote Access Trojan (RAT) were active on the npm registry.

The Attack Vector

The attackers leveraged the sudden influx of interest in Claude Code research to publish a compromised version of the popular `axios` HTTP library (versions 1.14.1 and 0.30.4). This malicious build included an unauthorized dependency on `plain-crypto-js`, which served as a payload delivery mechanism.

Infected Installations

Any user who initialized a new Claude Code environment or updated their dependencies during this three-hour window likely received the RAT-injected package. The trojan was designed to exfiltrate session tokens, SSH keys, and `.env` files from the host system.

Anthropic's Response

Anthropic responded by immediately deprecating the npm distribution channel for the Claude Code CLI. To protect the developer community, the company migrated to a native installer that bypasses npm entirely.

# Verified Native Installation Command
curl -fsSL https://claude.ai/install.sh | bash

Immediate Remediation

If your `package-lock.json` or `yarn.lock` references `axios` 1.14.1, 0.30.4, or `plain-crypto-js`, your system must be treated as fully compromised.

  • Rotate Credentials: All API keys, database passwords, and cloud access tokens must be rotated immediately.
  • System Wipe: Perform a clean reinstall of the operating system and restore data from a verified backup.
  • Audit Access: Review logs for any unauthorized logins or data exfiltration from your cloud providers.

The Claude Source Hub remains a safe environment for architectural research, but we urge all developers to prioritize supply-chain security by verifying hashes and pinning dependencies to known safe versions.
